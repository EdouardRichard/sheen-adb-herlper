# Sheen ADB 助手 v0.01 需求文档

> 状态：已确认（2026-07-11）  
> 产品名：Sheen ADB 助手（Sheen ADB Helper）  
> 本文定义 v0.01 的交付边界和验收标准；未写入本文的功能不属于本版本。

## 修订记录

- 2026-07-14：增加前台、进程内、强制脱敏的诊断事件查看能力，用于 Phase 0 及后续连接问题排查；不改变设备 Logcat 的采集规则。

## 1. 产品定位

Sheen ADB 助手是一款运行在安卓主控设备上的、纯本地的无线 ADB 调试工具。它让用户通过局域网连接另一台安卓设备，或通过 `127.0.0.1` 连接本机已开启的无线调试服务，以图形化方式查看设备状态、执行 Shell、查看进程和实时 Logcat。

v0.01 的目标是建立一个稳定、可解释、可恢复的“连接—诊断—调试”闭环，而不是做全功能设备管理器。

### 1.1 目标用户

- 熟悉开发者选项与 ADB 基本风险的安卓高级用户、开发者和设备维护人员。
- 需要用手机调试手机、平板、电视或盒子的用户。

### 1.2 非目标

- 不替代 Android Studio、电脑端 `adb` 或 MDM。
- 不绕过开发者选项、无线调试、RSA 授权或配对确认。
- 不提供 Root、提权、解锁、刷机、绕过系统保护或规避安全限制的功能。
- 不实现应用管理、文件管理、安装 APK、传输文件、高级重启、截图/录屏或后台日志采集。

## 2. 平台与兼容性

| 项目 | v0.01 约定 |
|---|---|
| 主控端 | Android 11（API 30）及以上；HyperOS 3 / Android 16 为首要验收基线 |
| 被控端 | Android 11+ 的手机、平板、电视、盒子，优先标准无线调试；旧式 ADB TCP/IP 为尽力兼容 |
| 最低 SDK | `minSdk 30` |
| 编译/目标 SDK | `compileSdk 36`、`targetSdk 36`（Android 16） |
| 网络 | 同一局域网或本机环回；不扫描局域网，不经由互联网中转 |
| 分发 | 小米应用商店；开源仓库与商店包功能等价 |

Android 11+ 的标准无线调试首次接入需要通过配对码或二维码建立可信主机身份，配对端口和调试连接端口不同。v0.01 仅实现配对码回退流程。参见 [Android 官方 ADB 文档](https://developer.android.com/tools/adb) 与 [AOSP ADB Wi-Fi 架构说明](https://android.googlesource.com/platform/packages/modules/adb/%2B/HEAD/docs/dev/adb_wifi.md)。

## 3. 不可违反的产品约束

- 纯本地：无账号、无后端、无云端同步、无广告、无统计/遥测 SDK。
- v0.01 永久免费：无内购、订阅、付费墙或预埋支付组件。
- 无 Root 依赖；不得使用无障碍服务自动操作开发者选项或无线调试设置。
- 权限最小化、按需请求；不申请所有文件访问权限，不申请自启动或无必要的常驻通知。
- ADB 私钥、配对材料、Shell 输出、Logcat 和设备信息默认不上传、不共享、不持久化。
- 所有危险后果必须在用户操作前以清楚文字说明；提示不改变用户输入的原始 Shell 命令。

## 4. 核心概念

| 名称 | 定义 |
|---|---|
| 主控端 | 安装并运行 Sheen ADB 助手的设备 |
| 被控端 | 运行 `adbd`、接受 ADB 连接的目标设备；本机模式下主控端与被控端为同一设备 |
| 设备档案 | 本地保存的显示名、地址、最近调试端口、连接时间和主机身份引用；不含日志内容 |
| 活跃会话 | 当前唯一允许执行 ADB 操作的已连接设备 |
| 配对端口 | Android 11+ 无线调试用于一次性配对码认证的端口 |
| 调试端口 | 配对成功后用于实际 ADB TLS 连接的端口 |

## 5. 信息架构与全局交互

### 5.1 主界面

主界面必须包含一个可编辑的地址输入框和相邻的“连接”按钮：

```text
☰  Sheen ADB 助手                       [未连接 / 设备名]

[ IP:端口                                                ][连接]
最近设备
```

- 地址解析支持 IPv4（`192.168.1.2:5555`）、主机名（`device.local:端口`）和带方括号的 IPv6（`[fe80::1]:端口`）。
- 不提供自动发现、mDNS 扫描或附近设备扫描。
- 连接中按钮显示进度并可取消；同一时间仅可存在一个活跃会话。
- 成功连接后自动创建或更新设备档案；切换设备时必须先断开旧会话。
- 顶部始终显示连接状态。未连接时，依赖 ADB 的菜单入口可见但置灰，并说明“请先连接设备”。

### 5.2 抽屉菜单

左上角三横线打开抽屉菜单，顺序固定：

1. 设备列表 / 首页
2. 设备概览
3. Shell 终端
4. 进程监控
5. Logcat
6. 设置与隐私

### 5.3 视觉与适配

- 简体中文优先；文案使用直白、可操作的中文。
- 跟随系统浅色/深色主题，使用简约 Material 3 视觉；颜色不可作为状态的唯一表达。
- 手机竖屏优先；平板或横屏使用双栏/扩展布局，不改变功能路径。
- 不依赖多指或隐藏手势；所有关键功能必须由可发现、可点击控件完成。
- 无障碍：交互控件具有语义标签，状态和错误可被读屏获取，文字支持系统字体缩放。

## 6. 连接与配对

### 6.1 直接连接（默认路径）

1. 用户输入调试 `IP:端口` 并点击连接。
2. 应用校验地址格式后发起 ADB 连接。
3. 若是已配对设备或旧式 ADB TCP/IP，连接成功后进入设备概览。
4. 若旧式 ADB 要求 RSA 授权，应用提示用户在被控端确认指纹，并允许重试。
5. 若识别为 Android 11+ 的认证/未配对失败，显示原因和“使用配对码”入口；不得把任意网络错误错误地说成需要配对。

### 6.2 配对码回退

仅在用户主动点击回退入口后显示配对页面：

- 输入：`IP:配对端口`、6 位配对码；主机地址预填为刚才直接连接的地址。
- 成功后，应用明确提示用户输入被控端“无线调试”主页面显示的调试端口，或在已知可用时自动连接。
- 配对成功立即保存主机身份；配对失败不得保存半成品档案或明文配对码。
- 本版本不请求相机权限、不实现二维码配对。

### 6.3 本机连接

- 抽屉提供“本机连接”入口，预填 `127.0.0.1`，用户仍需填写当前调试端口。
- 本机模式同样遵守系统的无线调试、配对与授权机制；应用不能自行开启无线调试，也不能保证重启后无操作恢复。
- 当系统不支持、端口不可达或与其他本地 ADB 工具冲突时，必须给出诊断性错误和手动处理建议。

### 6.4 连接状态与错误

连接状态只允许为：`未连接`、`连接中`、`等待设备授权`、`已连接`、`配对中`、`断开中`、`错误`。错误至少区分：地址无效、网络不可达、超时、TLS/认证失败、设备拒绝、协议不兼容、会话被远端关闭和未知错误。

每个错误页面必须同时包含：发生阶段、用户可理解的原因、可执行的下一步、技术详情的复制入口。不得泄露私钥、配对码或完整日志。

### 6.5 本地诊断事件

- 应用提供当前进程内的连接诊断事件查看入口，至少覆盖连接、配对、Shell 探针、断开、取消、失败和资源关闭。
- 诊断事件仅包含顺序、阶段、结果、结构化代码、脱敏目标和异常类型；禁止包含真实 IP/主机名、配对码、ADB 密钥/证书、原始命令、Shell 输出或设备 Logcat。
- 最多保留最近 100 条，用户可手动清空；退出进程后消失，不落盘、不上传、不调用 Android 系统日志，也不接入第三方日志或崩溃上报。

## 7. 功能需求

### 7.1 设备档案

- 配对或直接连接成功后自动保存：显示名、主机/IP、最近成功的调试端口、首次连接时间、最近连接时间、本机标记和主机身份引用。
- 用户可编辑显示名、删除单个档案、手动重连。
- 设备档案数量不设产品限制；列表按最近连接时间倒序。
- 删除档案必须同时删除其关联的主机身份材料，并二次确认。
- 不持久化 Shell 正文历史、终端输出、Logcat 内容或进程快照。

### 7.2 设备概览

显示以下尽力获取的信息，并为缺失字段标注“设备未提供”而非伪造默认值：

- 品牌、制造商、型号、设备代号、Android 版本、SDK、构建指纹/版本、安全补丁日期。
- CPU ABI、可用核心数、内存总量/可用量、内部存储总量/可用量。
- 电量、充电状态、温度（设备支持时）、运行时间、当前网络地址（设备支持时）。

静态信息首次进入时加载，用户可手动刷新；电池和内存在概览页面处于前台期间每 5 秒刷新。离开页面、切换设备或断开时必须停止轮询。

### 7.3 Shell 终端

- 支持多行原始 ADB Shell 命令、执行、运行中取消、标准输出/标准错误的可区分展示和复制。
- 不设命令白名单，不修改、不静默拦截用户命令。首次进入仅展示一次风险说明；遇到明显破坏性命令可提示风险，但用户可继续执行。
- 每条命令与对应输出按会话顺序展示；当前会话的内容仅在内存中保存。切换活跃设备或退出应用后清空。
- 默认命令执行上限为 30 秒；用户可在当前会话将该上限调整到 5 分钟。超时、取消和断开必须有明确结果标记。
- 终端可保留的输出总量上限为 1 MiB，达到上限时丢弃最早内容并显示提示。

### 7.4 进程监控

- 仅提供只读进程列表：进程名、PID、UID、状态及设备能够可靠提供的内存数据。
- 支持关键字搜索、手动刷新、复制 PID 或进程名。
- 不提供结束进程、Kill、强制停止或应用管理行为。
- 不同 ROM 的 `ps`/`top` 输出不兼容时，必须采用能力探测和降级解析；无法获取的字段留空并解释原因。

### 7.5 Logcat

- 仅在页面前台、用户明确点击“开始”后抓取；离开页面、断开或点击停止时立即停止。
- 默认缓冲区：`main`、`system`、`crash`；默认最低等级：`Info`；默认格式：线程时间。
- 支持选择 `Verbose` 至 `Fatal` 等级、选择 `main/system/crash/radio/events` 缓冲区、关键字过滤、暂停/继续、清屏、复制可见内容。
- 缓冲区上限为 **10,000 行或 4 MiB（先达到者）**；丢弃最早记录并有可见提示。
- 用户可通过系统文件创建器手动导出当前可见内容为 UTF-8 `.txt`；不使用所有文件访问权限，不自动保存或上传。

### 7.6 设置与隐私

- 显示应用版本、开源许可证、第三方依赖许可证、隐私承诺、风险说明和支持的系统范围。
- 提供“清除所有本地数据”，删除设备档案、关联密钥、偏好和临时数据；操作必须二次确认。
- 提供打开系统开发者选项/无线调试页面的帮助入口（若系统 Intent 不可用，则展示手动路径）；不得自动更改任何系统设置。

## 8. 权限、数据与安全

| 项目 | 规则 |
|---|---|
| 网络 | 仅为局域网 ADB 连接申请最小必要网络能力；Android 16 相关本地网络限制须按系统要求适配 |
| 文件导出 | 使用 Storage Access Framework 的用户选择路径，不申请所有文件访问 |
| 相机/定位/通讯录等 | v0.01 禁止申请 |
| ADB 主机身份 | 仅本机保存，使用 Android Keystore 保护；禁止写入日志、分享内容或崩溃报告 |
| 配对码 | 只在内存中短暂使用，成功/失败/页面离开后立即清除 |
| Shell 与 Logcat | 默认不落盘、不上传；只有用户手动复制或导出才离开应用内存 |
| 设备信息 | 仅用于当前界面和本地档案显示，不上传 |

小米应用商店要求权限使用满足最小必要、用途透明并在审核资料中说明；本应用须始终遵循该原则。[小米权限使用及申请标准](https://dev.mi.com/xiaomihyperos/documentation/detail?pId=2156)

## 9. 非功能要求

- 任何耗时操作必须显示进行状态、可取消路径或合理的超时提示；不得阻塞主线程或产生 ANR。
- 网络、认证、协议、解析和存储错误必须转换为用户可理解的状态，不得使页面崩溃。
- 所有 ADB 资源（Socket、流、协程、观察任务）在取消、断开、切换设备和页面销毁时关闭。
- 应用在无网络、设备掉线、端口变更、用户拒绝授权和 ROM 命令输出差异下可恢复使用。
- 不显示“已连接”除非 ADB 握手和基础命令验证完成。

## 10. 验收标准

### 10.1 连接闭环

1. 用户可从首页直接连接已配对的 Android 11+ 设备，并在成功后自动看到设备档案。
2. 新 Android 11+ 设备直接连接失败时，仅在认证/配对相关错误下显示配对码回退；配对后可使用调试端口连接。
3. 本机 `127.0.0.1` 在用户手动开启无线调试并完成必要配对后，可执行 `echo` 并主动断开。
4. 旧式 `IP:5555` 设备在其允许时可连接；不可用时显示可行动的错误，而非假称已支持。
5. 切换设备后，旧设备不能再接收 Shell 或 Logcat 操作。

### 10.2 调试闭环

1. 设备概览正确加载可获得字段，动态指标仅在页面可见时按约定刷新。
2. Shell 能展示输出、取消和超时，且退出/切换后不保留命令历史。
3. 进程页面可搜索、刷新、复制，且没有终止进程入口。
4. Logcat 的默认值、过滤、暂停、容量丢弃和系统文件导出均可验证。

### 10.3 分发与隐私闭环

1. Release 包不包含广告、统计、支付、云端请求、Root 或无障碍自动化能力。
2. 不申请不必要危险权限或所有文件访问权限。
3. 清除全部本地数据后，设备档案和关联密钥均无法恢复使用。
4. 开源仓库中不存在签名密钥、真实设备数据、配对码或日志样本。
