# Sheen ADB 助手：权限矩阵

> 状态：v0.01–v0.04 当前实现事实（v0.04 merged Manifest 复核：2026-07-23）
> 分发：默认不面向应用商店上架，见 [`adr/0005-distribution-strategy.md`](adr/0005-distribution-strategy.md)。
> 规则：仅“当前允许”项可进入 Manifest。任何新增、启用或用途变更均需修改此表、需求文档和相应 ADR。

| 权限/能力 | Manifest 状态 | 用户可见用途 | 请求时机/降级 | 审核与验证 |
|---|---|---|---|---|
| `android.permission.INTERNET` | **已声明**（普通权限） | 与用户手动输入或从已公布 NSD 服务中选择的本机/局域网 ADB 地址建立 TCP/TLS 连接 | 安装时声明；没有网络时仅显示结构化连接失败与排查建议 | 无产品后端、不探测未公布服务；以 merged manifest、连接错误映射和真机远端/环回连接验证 |
| `android.permission.ACCESS_NETWORK_STATE` | **已声明（v0.04，普通权限）** | 识别当前主控端 `Network`，把 ADB NSD 发现绑定到当前网络，并在网络切换时停止旧发现 generation | 安装时声明，无运行时弹窗；只在用户进入无线发现或本机配对时读取，失败时停止自动发现并保留手动地址 | merged Manifest 已确认；不读取 SSID/BSSID/位置。网络切换、IPv4/IPv6 真机矩阵仍待执行；见 ADR 0007 |
| `android.permission.CHANGE_WIFI_MULTICAST_STATE` | **已声明（v0.04，普通权限）** | API 30–32 在用户发起 ADB NSD 发现期间临时持有 Wi-Fi 组播锁，以可靠接收系统公布的无线调试服务 | 安装时声明，无运行时弹窗；仅活动发现窗口获取，停止/取消/超时/后台/网络切换均释放；不可用时保留手动地址 | merged Manifest 与自动化 lock 清理已确认；不用于广播或端口扫描。Android 11/12 真机仍待执行；见 ADR 0007 |
| Storage Access Framework 系统文件选择器 | **已使用；v0.03 计划扩展，仍无 Manifest 权限** | 用户手动导出当前可见 Logcat；v0.03 由用户选择单个上传来源及下载/APK 保存目录 | Logcat 使用 `CreateDocument`；v0.03 上传使用 `OpenDocument`，下载和 APK 提取使用 `OpenDocumentTree`；取消即不读写，授权不持久化 | 只访问用户明确选择的 URI/目录树；输出先暂存后提交，不扫描树外文件，不缓存完整副本；Android 11+ 根目录限制明确降级 |
| `android.permission.NEARBY_WIFI_DEVICES` / 本地网络运行时权限 | **不声明** | v0.04 targetSdk 36 使用系统 NSD 发现当前网络已公布的 ADB TLS 服务，无需 Nearby、位置或未来 targetSdk 的本地网络运行时权限 | 不请求；受 ROM、VPN、网络隔离或未来平台策略限制时保留手动连接 | targetSdk/平台策略变化必须重新走规格、权限矩阵与 ADR，禁止预埋；见 ADR 0007 |
| 被控端应用列表与单包管理 | **无需新增 Manifest 权限** | 通过用户已建立的 ADB Session 读取当前用户第三方包，并执行已确认的强停、禁用或重新启用 | 仅在已连接且用户进入应用管理/确认单包操作时使用；失败时不影响已有只读调试功能 | 不读取主控端应用列表；不声明 `QUERY_ALL_PACKAGES`；以 merged manifest、命令边界测试和真机当前用户范围验证 |
| 相机 | 禁止 | v0.04 二维码由主控端显示、被控端系统无线调试扫描，本应用不扫描 | 不请求；二维码不可用时使用配对码 | merged Manifest 不含相机；QR 角色与临时材料清理由自动化/真机验证 |
| 所有文件访问 / 媒体读取 | 禁止 | v0.03 文件传输和 APK 提取不需要主控端全盘或媒体库访问 | 不请求；用户通过 SAF 明确选择来源与目标，provider 能力不足时更换位置 | 禁止 `MANAGE_EXTERNAL_STORAGE`、存储/媒体权限和直接扫描；以 merged Manifest 与 SAF 边界测试验证 |
| 定位、蓝牙、通讯录、电话、短信、日历、麦克风 | 禁止 | 与 v0.01 无关 | 不适用 | 严格最小权限 |
| `android.permission.POST_NOTIFICATIONS` | **已声明（v0.04；Android 13+ 运行时）** | 本机无线调试模式发现配对端口后，在通知内输入六位配对码 | 仅用户进入本机通知配对时请求；拒绝、通知关闭或 OEM inline reply 不兼容时回退应用内输入，并提示可尝试原生通知样式 | merged Manifest、权限拒绝和锁屏策略自动化已确认；通知不含码、真实端点或设备身份。Android 13–16/OEM 真机仍待执行；见 ADR 0007 |
| `android.permission.FOREGROUND_SERVICE` / `shortService` | **已声明（v0.04，普通权限 + 非导出组件）** | 用户从本应用切换到系统无线调试设置时，最长 2 分钟维持本机端口发现与配对通知 | Activity 可见时启动，5 秒内前台；成功、取消、端口失效、Session 变化或 2 分钟到期立即停止；通知拒绝不扩大时限 | merged Manifest 已确认唯一 Service 为 `exported=false`、`foregroundServiceType=shortService`；不承载 LAN、文件、应用元数据或 Logcat。系统超时真机仍待执行；见 ADR 0007 |
| 自启动、开机广播、忽略电池优化 | 禁止 | v0.04 短服务不需要常驻或开机恢复；v0.03 文件/Logcat 仍只按既有前台生命周期 | 不请求；进程终止或窗口到期即结束 | Manifest 不含 boot receiver/电池豁免；不得用本功能延长其他任务 |
| 无障碍服务、设备管理员、Root/SU | 禁止 | 不自动操作系统设置，不提权 | 不适用 | 仅使用用户在系统无线调试中授予的 ADB 能力 |

## 变更检查清单

新增或修改权限前，提交者必须回答：

1. 哪一项已批准的用户需求在没有该权限时无法完成？
2. 是否存在系统 Intent、SAF、用户手动输入或无权限实现？
3. 权限在何时请求、拒绝后如何降级、何时不再使用？
4. 当前是否存在项目负责人明确批准的分发渠道？如有，该渠道的权限声明、隐私政策、演示和测试步骤如何更新？
5. 该变更是否需要 ADR，并是否影响“纯本地、最小权限”的不可变约束？
