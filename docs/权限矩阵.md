# Sheen ADB 助手：权限矩阵

> 状态：v0.01 已确认（2026-07-11）  
> 规则：仅“当前允许”项可进入 Manifest。任何新增、启用或用途变更均需修改此表、需求文档和相应 ADR。

| 权限/能力 | v0.01 状态 | 用户可见用途 | 请求时机/降级 | 审核说明 |
|---|---|---|---|---|
| `android.permission.INTERNET` | 当前允许（普通权限） | 与用户输入的本机或局域网 ADB 地址建立 TCP/TLS 连接 | 安装时声明；没有网络时仅显示连接失败与排查建议 | 本应用没有后端；网络仅用于用户主动发起的 ADB 连接 |
| `android.permission.ACCESS_NETWORK_STATE` | 当前允许（普通权限） | 在连接失败时判断网络是否可用，减少误导性错误 | 安装时声明；读取失败时不影响用户手动连接 | 只读取基础连通状态，不采集或上传网络信息 |
| Storage Access Framework 系统文件选择器 | 当前允许（无存储权限） | 用户手动导出当前可见 Logcat 文本 | 点击“导出”后由系统选择位置；取消即不写入 | 不使用 `MANAGE_EXTERNAL_STORAGE`，应用不遍历或读取用户文件 |
| `android.permission.NEARBY_WIFI_DEVICES` | 暂不声明；条件性评审 | 仅当 Android 本地网络保护在目标系统实际拦截直连 ADB 且无更小替代方案时，用于恢复本地网络访问 | 必须先经 ADR、真机复现、审核材料更新和用户按需授权；拒绝时保留手动连接界面并说明限制 | Android 平台的本地网络策略仍在演进；不可为预防性便利而提前申请 |
| 相机 | 禁止 | 二维码配对不在 v0.01 | 不适用 | v0.01 仅配对码；未来扫码须独立评审 |
| 所有文件访问 / 媒体读取 | 禁止 | 文件管理、APK、日志目录访问均不在 v0.01 | 不适用 | 日志导出只用系统文件选择器 |
| 定位、蓝牙、通讯录、电话、短信、日历、麦克风 | 禁止 | 与 v0.01 无关 | 不适用 | 严格最小权限 |
| 通知、前台服务、自启动、开机广播、忽略电池优化 | 禁止 | v0.01 无后台任务、无持续日志 | 不适用 | 前台实时页面离开即停止 |
| 无障碍服务、设备管理员、Root/SU | 禁止 | 不自动操作系统设置，不提权 | 不适用 | 仅使用用户在系统无线调试中授予的 ADB 能力 |

## 变更检查清单

新增或修改权限前，提交者必须回答：

1. 哪一项已批准的用户需求在没有该权限时无法完成？
2. 是否存在系统 Intent、SAF、用户手动输入或无权限实现？
3. 权限在何时请求、拒绝后如何降级、何时不再使用？
4. 小米应用商店的权限声明、隐私政策、截图/演示和测试步骤如何更新？
5. 该变更是否需要 ADR，并是否影响“纯本地、最小权限”的不可变约束？
