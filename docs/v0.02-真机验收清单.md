## 1. 测试矩阵

- 主控端：Android 11/API 30；HyperOS 3/Android 16。
- 被控端：Android 11；HyperOS 3/Android 16；至少一台非小米 Android 14+。
- 形态：手机；平板；电视或盒子至少一种。
- 连接：远端无线调试；本机 `127.0.0.1`；设备明确支持时的旧式 `:5555`。
- 用户：主用户；工作资料或次用户至少完成一次负向验证。
- 脱敏样本：普通第三方、前台运行、后台运行、已禁用、系统预装、受策略管理应用。记录中统一写“样本 A/B/C”，不得写真实包名。

## 3. 安全边界与确认

- [ ] `APP-DEVICE-011` 每次禁用都出现新确认框，说明入口、通知、后台任务和关联功能风险。

- [ ] `APP-DEVICE-014` 系统预装样本、状态未知样本和列表外目标均无修改入口。

## 4. 强制停止

- [ ] `APP-DEVICE-017` 前台第三方样本确认强停后显示“已发送请求”语义，不宣称永久停止。
- [ ] `APP-DEVICE-018` 后台第三方样本强停不影响应用连接管理器后续刷新或重试。
- [ ] `APP-DEVICE-019` 确认期间卸载目标后返回 `ADB_APP_PACKAGE_NOT_FOUND` 并引导刷新。
- [ ] `APP-DEVICE-020` ROM/企业策略拒绝强停时返回 `ADB_APP_POLICY_REJECTED`，不显示成功。

## 5. 禁用与重新启用

- [ ] `APP-DEVICE-021` 已启用第三方样本禁用后重新读取状态，只有确认为已禁用才显示验证成功。
- [ ] `APP-DEVICE-022` 已禁用第三方样本重新启用后重新读取状态，只有确认为已启用才显示验证成功。
- [ ] `APP-DEVICE-023` 修改命令返回但状态不符合预期时显示 `ADB_APP_STATE_VERIFY_FAILED`，不误报成功。
- [ ] `APP-DEVICE-024` 受策略管理样本拒绝禁用/启用时显示 `ADB_APP_POLICY_REJECTED` 并保留其他调试能力。

## 6. 超时、取消、掉线与 session 切换

- [ ] `APP-DEVICE-026` 列表读取超过 15 秒时可取消或超时，临时命令/流已清理，页面可安全重试。
- [ ] `APP-DEVICE-027` 强停超过 10 秒、主动取消或掉线时显示结果未知，并要求重新连接/刷新。
- [ ] `APP-DEVICE-028` 状态修改与验证合计超过 15 秒、主动取消或掉线时显示结果未知，不宣称已回滚或未执行。
- [ ] `APP-DEVICE-029` 加载或修改期间其他应用管理按钮禁用，取消入口可发现、可读屏操作。
- [ ] `APP-DEVICE-030` 修改过程中关闭无线调试、目标重启或端口变化后，连接状态不继续显示已连接。
- [ ] `APP-DEVICE-031` 从应用管理离开时列表读取取消、确认框清除；已经发出的修改结果不会写回离开的旧页面状态。
- [ ] `APP-DEVICE-032` 操作期间切换到另一设备，旧 session 结果、列表、筛选、确认和通知均不进入新设备 UI。
- [ ] `APP-DEVICE-033` 单次失败后重新连接可再次加载和操作，不破坏会话管理器重试能力。

## 8. ROM 风险记录

人工验收若失败，只记录对应 ID 与以下脱敏分类：

- `ROM-CURRENT-USER`：当前用户命令缺失、噪声或语义不同。
- `ROM-PACKAGE-LIST`：`pm`/`cmd package` 列表参数或输出差异。
- `ROM-ENABLED-STATE`：禁用列表或 enabled state 语义不同。
- `ROM-POLICY`：企业策略、系统保护或厂商安全中心拒绝。
- `ROM-SESSION`：无线调试端口变化、EOF、TLS/Socket 取消清理差异。
- `ROM-UI`：电视/盒子焦点、平板或大字体适配差异。

不得把 ROM 不支持、策略拒绝、状态未知或未执行真机验证记录为通过。

## 9. Shell 与 Logcat 回归复测

- [ ] `REG-DEVICE-001` 在同一远端 session 连续执行多条有限时 Shell 命令；成功与非零退出均只结束本次命令流，后续命令无需重新连接。
- [ ] `REG-DEVICE-002` Shell 命令超时后连接仍可直接执行下一条命令，错误为 `ADB_TIMEOUT`，不显示 `ADB_UNKNOWN`。
- [ ] `REG-DEVICE-003` 主动取消 Shell 命令后连接仍可直接执行下一条命令；普通设备端命令保持原样。输入电脑端 `adb shell` 包装写法时必须先展示将执行的设备端内容，并分别验证“执行设备命令”和“仍原样执行”，不得静默改写。
- [ ] `REG-DEVICE-004` 设备端提前关闭命令子流时显示 `ADB_COMMAND_STREAM_CLOSED`；普通子流 I/O 异常显示 `ADB_IO_FAILURE`，均不误关仍有效的底层连接。
- [ ] `REG-DEVICE-005` Logcat 启动后跨越至少一次 30 秒无匹配日志的空闲期仍保持采集，新日志到达后继续显示。
- [ ] `REG-DEVICE-006` 分别切换最低等级、缓冲区和关键字，界面始终只显示当前条件下最新 100 条；第 101 条到达后最旧可见记录被移除。
- [ ] `REG-DEVICE-007` 暂停期间界面内容不变化；继续后补齐到当前过滤结果的最新 100 条；清屏后无需重启采集即可接收新日志。
- [ ] `REG-DEVICE-008` 依次验证停止、离开页面、取消、断开和 session 切换：旧采集立即结束，旧日志与错误不进入新 session 页面，相关子流和连接资源按归属释放。
- [ ] `REG-DEVICE-009` 采集过程不产生自动文件、上传或包含日志内容的诊断记录；仅用户主动使用 v0.01 既有导出动作时写入其选择的文档位置。
