# 应用管理现状

> 类型：当前架构事实
> 依据：HEAD 的 `:core:adb`、`:feature:apps`、`:app` 及对应测试。

## 当前能力

- `:feature:apps` 先显示被控端当前 Android 用户的第三方应用包列表，再按当前 Session 逐项补齐应用名与图标。名称可用时以名称为主标题且始终显示唯一包名；缺失时保留包名和统一占位图。
- 搜索对 trim 后的应用名或包名做大小写不敏感包含匹配，两字段为 OR；全部/已启用/已禁用筛选与搜索取交集。元数据到达时结果即时重算，不轮询、不持久化。
- 可靠可得时同时显示版本/安装器字段；名称、图标或其他字段不可可靠获得时以明确 metadata 状态降级，不猜测、不隐藏该包。
- 用户可对单个允许目标发起强制停止、禁用或重新启用；每次修改均需一次性确认。

## 核心与安全边界

- 当前用户探测、第三方包/UID 列表、包名校验、命令构造、Session 快照、APK 路径与 Sync 读取、元数据解析、修改和后置验证都位于 `:core:adb`。
- Feature 只接收项目自有 `RemoteApplication`、`ApplicationMetadataUpdate` 和受限图标载荷；不得提供 APK 路径、拼接 Shell、访问 Sync stream 或接触第三方 parser 类型。
- 修改目标必须来自同 Session 最近的安全快照，属于当前用户且非系统包；包名采用保守语法并有容量上限。
- 本机 `127.0.0.1` Session 禁止操作 Sheen 自身；未知状态、系统包、列表外或过期 Session 目标均拒绝。
- 强停只返回“请求已接受”语义；启用状态修改只有后置查询符合预期才返回已验证成功。
- 修改后超时、取消、EOF 或掉线可能产生结果未知；UI 要求重新连接/刷新，不宣称成功、失败或回滚。

## 状态、数据与限制

- 列表、当前用户、名称、图标、搜索/筛选、确认和操作结果只在内存中；断开、取消或 Session 变化清空旧状态并拒绝旧 metadata update。
- 元数据加载单 APK 输入上限 32 MiB、单图标编码上限 1 MiB、图标 LRU 总计 16 MiB、整批默认 10 秒并顺序处理；APK 字节只通过当前 Session 的有界 Sync reader 进入内存，完成后释放且不落盘。
- `apk-parser 2.6.10` 只存在于可替换的核心内部 adapter；ZIP traversal、压缩膨胀、损坏资源、split-only、超限、权限和解析失败均返回项目自有降级状态。自动化恶意输入边界已有测试，Android API 30 真机 runtime smoke 仍须按 v0.04 验证矩阵完成。
- 诊断事件不含包名、用户 ID、命令、APK 路径或输出；应用元数据不新增 Manifest 权限或 DataStore 格式。
- 不支持安装、卸载、系统应用、跨用户、批量、定时、后台、Root、Shizuku、设备管理员或无障碍自动化。
