# 设备与本地数据现状

> 类型：当前主干事实
> 依据：HEAD 的 `:core:data`、`:core:adb`、设备/概览/设置 Feature 及测试。

## 模块归属

- `:core:data` 拥有设备档案模型、DataStore 仓库、档案变更、文本导出与临时数据清理。
- `:feature:devices` 负责手动地址、二维码/六位码选择、本机配对、局域网发现、连接替换确认、最近设备、重命名、删除和进程内诊断展示。
- `:core:adb` 负责无线发现、QR/六位码配对、本机窗口、端点复核和单 Session 所有权；Feature 不接触 `NsdManager`、Socket、Kadb 或原始端点拼接。
- `:feature:overview` 通过 `AdbSessionManager` 加载设备概览，并只在前台刷新动态指标。
- `:feature:settings` 展示版本、许可证/隐私信息、系统设置帮助和清除全部本地数据入口。

## 数据归属

- 设备档案保存稳定 ID、显示名、主机、调试端口、首次/最近连接时间、本机标记和身份引用；按最近连接排序。
- 档案元数据保存在 DataStore；损坏的版本化记录被拒绝，不解释为有效档案。
- ADB 主机身份不属于 DataStore，由 `:core:adb` 管理；最后一个相关档案删除或清除全部数据时清除身份。
- 概览、进程快照、Shell、Logcat、应用列表和操作结果不进入档案或 DataStore。
- QR 材料、六位码、NSD 服务观察、真实发现端点、local pairing token 与通知输入均为当前进程/窗口内临时状态，不写入设备档案或 SavedState。

## 公共行为与限制

- 概览尽力提供系统、构建、CPU、内存、存储、电池、温度、运行时间和网络字段；不可可靠获得的字段保持缺失，不伪造默认值。
- 动态指标由页面前台状态控制刷新；离开、断开或切换 Session 后停止。
- 局域网发现只在设备发现页面前台的有界窗口内运行，离页/后台/取消/网络切换即停止；未发现或受限时保留手动地址，不改为端口扫描。
- 本机模式是唯一允许短暂跨设置页面继续的发现路径，由非导出 `shortService` 维持且最长 2 分钟；它不延长 LAN、文件、应用元数据或 Logcat 生命周期。
- 配对不会自动连接。选择连接其他设备且已有 Session 时，UI 必须取得显式替换确认；发现条目在用户选择后由核心重新确认，过期端点返回非成功。
- 文本导出写入用户选择的 SAF 目标，不申请所有文件访问，也不缓存导出副本。
- 清除全部本地数据覆盖档案、身份、偏好和临时文件；产品不提供账号或云端恢复。

## 安全边界

- 主机地址属于本地元数据，日志和测试输出必须脱敏；设备详情和身份材料不得上传。
- 当前无线发现与本机通知只使用权限矩阵批准的网络状态、临时组播、通知和短时前台服务能力；不声明相机、位置、Nearby、存储、开机或电池豁免权限。最终权限事实以 [`docs/权限矩阵.md`](../权限矩阵.md) 与 merged Manifest 为准。
- NSD-only 范围、通知锁屏降级和 2 分钟生命周期见 [ADR 0007](../adr/0007-wireless-discovery-and-local-pairing-lifecycle.md)；Android 11–16 与 OEM 真机证据尚未完成时不得写成已验证。
