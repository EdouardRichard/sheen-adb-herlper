## v0.04 候选依赖许可证与接入门禁（T001，2026-07-22）

下表登记接入前的静态供应链事实；当前没有因此新增 Gradle 依赖。发布 POM 是依赖图和许可证声明的权威来源，仓库 LICENSE/归档状态用于交叉核实。未实际执行的 Android 或恶意输入检查必须保留为待执行门禁。

| 坐标 | 精确版本 / 许可证 | 受限用途 | 传递依赖处理 | 维护、替代与移除 | 当前门禁状态 |
|---|---|---|---|---|---|
| `com.google.zxing:core` | `3.5.4` / Apache-2.0（白名单内） | 仅由 `:feature:devices` 的项目自有 `QrMatrixEncoder` 编码 QR；没有 scanner、camera integration 或 JavaSE artifact。 | POM 仅有 `junit:junit` 的 test scope；运行时无传递依赖，禁止把该测试项作为运行时依赖加入。 | 上游未归档，3.5.4 于 2025-11-11 发布；中等维护风险（无路线图承诺）。替代为自有 QR 编码器；移除时删除 ZXing 适配器、catalog alias 和 `:feature:devices` implementation，并重跑互操作测试。 | **条件通过，可接入 T002/T003。** Gradle 图、Debug 编译与 QR 互操作尚未实际验证，不能声称已通过 Android runtime。 |
| `net.dongliu:apk-parser` | `2.6.10` / BSD-2-Clause（白名单内） | 仅在 `:core:adb` 内部、可替换的 APK 元数据适配器解析 label、资源和图标；不得向公开模型泄漏第三方类型。 | POM 中 `jsr305:3.0.0`、`bcprov-jdk15on:1.58`、`bcpkix-jdk15on:1.58` 均为 `optional=true`，`junit:4.12` 为 test scope。Gradle 不传递加入 optional 项；默认不显式添加。若未来需要任一 optional 项，必须先分别完成许可证白名单、固定版本和解析图审查。 | 上游已归档，**高维护风险**。替代为自有受限 Manifest/resources parser，或返回 Plan 选择重新审查的已维护实现；移除时删除内部适配器、catalog alias 和 `:core:adb` implementation，保留自有模型/有界 reader。 | **高风险、仅条件接入；US4 为 BLOCKED。** 只有实际通过恶意 ZIP 防护（traversal、压缩炸弹、损坏 ZIP、资源边界）及 Android API 30 runtime smoke test 后才能解除。任一失败即保持 US4 `BLOCKED` 并返回 Plan 选替代 parser；不得用整项 `UNSUPPORTED` 或仅包名占位宣称应用名/图标需求完成。 |

### 证据状态与发布边界

- Android API 30 runtime、恶意 APK/ZIP、Gradle 解析和体积测量均尚未执行，状态为 `NOT_RUN`；不得以 POM、LICENSE 或样例 APK 解析结果伪造这些结论。
- 新依赖门禁、功能完成和设备验证均不解除既有发布阻断：在 ADR 0004 规定的条件完成前，`spake2-java 1.0.5` 的 GPL-3.0-or-later 仍使发布合规为 **BLOCKED**。

## 结项复核说明

- `spake2-java 1.0.5` 的发布 POM 与[精确标签 LICENSE](https://github.com/Flyfish233/spake2-java/blob/1.0.5/LICENSE) 均指向 GPL-3.0-or-later；原 Apache-2.0 记录已更正。
- 当前应用内许可证页面仍沿用原记录，本轮遵守“只做结项文档、不修改业务代码”的边界，未修改该页面。许可证冲突解决后，应用内展示、NOTICE 和本文必须一并复核。
- 处置方案与完成门禁见 [`adr/0004-spake2-license-compliance.md`](adr/0004-spake2-license-compliance.md)。
