# Phase 0：HyperOS 3 真机验收清单

> 当前状态：**待执行**。自动化测试和 Debug APK 构建通过不等于无线 ADB 真机 PoC 已验收。执行时不要把真实 IP、配对码、Shell 输出或设备日志写入仓库、Issue 或截图附件。

## 准备

- 主控端与被控端均为 Android 11+；首要基线为 HyperOS 3 / Android 16。
- 两台设备连接同一可信局域网；本机模式只需一台设备，但系统无线调试仍可能要求 Wi-Fi。
- 在被控端由用户手动打开开发者选项和无线调试。应用不会也不能替用户开启。
- 本机配对时建议把“无线调试”设置页与 Sheen ADB 助手置于分屏/小窗，避免切走后 6 位配对码弹窗失效。

## 首次远端配对与连接

1. 被控 HyperOS 3：在设置中搜索“开发者选项”；若尚未启用，进入“我的设备/关于手机”，连续点击 OS 版本，按系统提示完成启用。
2. 进入“更多设置 → 开发者选项 → 无线调试”，由用户打开无线调试并确认当前网络。
3. 先记录无线调试主页面的“IP 地址和端口”（这是**调试端口**）。
4. 在 PoC 诊断页原样输入第 3 步记录的动态调试地址并点“连接”；不要把 Android 11+ 标准无线调试端口替换成固定的 `5555`，该端口仅用于被控端已明确启用的旧式 ADB TCP/IP。
5. 只有出现 `ADB_AUTH_FAILED` 时才点“使用配对码”。`ADB_NETWORK_UNREACHABLE`（包括端口拒绝连接）、超时或协议错误应先按页面建议核对调试端口，不应进入配对流程。
6. 被控端点“使用配对码配对设备”，保持弹窗可见；把弹窗中的地址/端口（这是**配对端口**）和 6 位配对码填入应用。
7. 点“配对”。字段应立即清空；成功后页面提示返回使用调试端口。
8. 重新在主输入框填写第 3 步的调试地址并连接。只有 ADB 握手与内部基础探针成功后才应显示“已连接”。
9. 点“运行 echo sheen-poc”。期望：退出码 `0`，标准输出为 `sheen-poc`（末尾可有换行），标准错误为空。
10. 点“断开连接”。状态应回到“未连接”，当前命令输出从页面内存清除。

## 本机 `127.0.0.1` 验收

1. 在同一台 HyperOS 3 设备上用分屏/小窗保持无线调试设置页可见。
2. 若尚未配对，按上述流程使用配对弹窗；配对地址主机可尝试 `127.0.0.1`，端口必须使用弹窗显示的配对端口。若 ROM 只接受设置页显示的局域网地址，使用该地址完成配对，不应将差异伪装为成功。
3. 点“本机 127.0.0.1”，手动补上无线调试主页面的调试端口。
4. 连接并运行 PoC 命令，期望结果同上，然后主动断开。

## 必做负向验证

- 地址：IPv4、hostname、`[IPv6]:端口`；缺端口、裸 IPv6、0/65536 端口应在联网前拒绝。
- 网络不可达：错误必须是网络/端口诊断，不显示配对入口。
- 错误配对码：失败后配对码字段清空，不创建“已连接”状态。
- 配对端口误作调试端口：应显示认证或协议错误，不显示成功。
- 连接中、配对中和 Shell 执行中分别点取消：UI 明确显示未连接/已取消，Socket 最终关闭，可立即重试。
- 连接超时、关闭无线调试、端口变化、被控端掉线：状态可恢复，不崩溃、不把旧会话结果写回新会话。
- 技术详情复制：只出现结构化代码、异常类型和 `<地址已脱敏>`；不得包含真实 IP、配对码、证书、私钥或 Shell 输出。
- 页面“查看诊断日志”：可看到连接、配对、取消、失败和资源关闭事件；最多 100 条且可清空，内容不得出现真实地址/主机名、配对码、原始命令、Shell 输出或设备 Logcat，杀掉进程后不保留。
- 进程退出后检查应用私有目录：ADB RSA 私钥只存在为 Android Keystore AES-GCM 包装的密文；系统备份/设备迁移不包含身份文件。
