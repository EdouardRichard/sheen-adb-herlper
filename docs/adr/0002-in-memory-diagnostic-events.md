# ADR 0002：进程内脱敏诊断事件

- 状态：采用
- 日期：2026-07-14
- 负责人：Sheen ADB 助手项目

## 背景

Phase 0 真机验证出现连接失败时，页面只能展示最后一个结构化错误，无法确认连接、认证、资源关闭和取消的先后顺序。直接启用 Kadb、Android Logcat 或文件日志会扩大敏感数据暴露面，并违反 ADB 私钥、配对码、真实地址、Shell 输出和设备日志不得记录的约束。

## 候选方案

| 方案 | 优点 | 风险/结论 |
|---|---|---|
| 启用 Kadb/Android 原始日志 | 细节多 | 可能包含协议与设备敏感数据，Release 也难以证明彻底关闭；不采用 |
| 写入应用私有文件 | 可跨进程保留 | 扩大持久化、清理和导出边界；Phase 0 不需要；不采用 |
| 进程内结构化事件环形缓冲区 | 可查看操作时序，数据面最小，退出即清除 | 需要严格限制字段；采用 |

## 决策

由 `:core:adb` 的 `AdbSessionManager` 生成项目自有的结构化诊断事件，Feature/UI 只读展示：

1. 每条事件只允许包含单调递增序号、操作阶段、结果、项目技术代码、脱敏目标与异常类名；
2. 禁止传入异常消息、真实端点、配对码、密钥/证书、原始命令、Shell 输出和设备 Logcat；
3. 缓冲区固定为最近 100 条，用户可清空，进程退出后自然销毁；
4. 不写文件、不调用 Android `Log`、不上传、不复制到崩溃报告，也不打开 Kadb 的 `KADB_LOGGING`；
5. 该能力不新增权限、依赖或后台任务。

## 理由

结构化白名单字段比对任意日志文本做事后正则脱敏更容易审查和测试，同时足以定位错误映射、状态顺序、取消和资源清理问题。环形内存缓冲区限制了驻留时间与容量。

## 影响

- `:core:adb` 对外增加只读诊断事件流和清空操作，仍由会话管理器独占产生事件。
- `:app` PoC 页面增加查看与清空入口，不接触 Socket、TLS、密钥或原始 ADB 命令。
- 自动化测试必须验证容量上限、清空，以及端点、命令和输出不会进入事件。
- 该日志不是被控设备 Logcat，也不替代 v0.01 的前台 Logcat 功能。

## 回滚方式

移除 `AdbSessionManager.diagnosticEvents`、事件模型和 PoC 展示卡即可；没有磁盘格式、迁移数据、权限或第三方依赖需要清理。
