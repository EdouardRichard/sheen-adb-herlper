## v0.04 候选运行时依赖门禁（T001，2026-07-22）

本节是接入前的静态供应链审查，不表示依赖已写入 Gradle，也不以计划替代构建、恶意输入或真机证据。许可证、坐标与依赖图来自精确版本的发布 POM；维护状态和许可证以各上游仓库为准。

| 坐标 | 精确版本与许可证 | 唯一用途和模块边界 | POM 依赖图与处理 | 维护状态、替代与移除路径 | 接入/功能门禁 |
|---|---|---|---|---|---|
| `com.google.zxing:core` | `3.5.4`，Apache-2.0（白名单内） | 仅在 `:feature:devices` 通过项目自有 `QrMatrixEncoder` 编码 QR；不引入 ZXing Android scanner、camera integration 或 JavaSE artifact。 | 发布 POM 仅列未定版本的 `junit:junit` **test** 依赖；运行时图为 `core` 本身，无运行时传递依赖。不得为测试项补入运行时配置。 | 上游仓库未归档，`zxing-3.5.4` 于 2025-11-11 发布；维护风险为中等（上游未承诺路线图）。替代为项目自有 QR 编码器；移除时删除 `QrMatrixEncoder` 的 ZXing 适配器、版本 catalog alias 与 `:feature:devices` implementation，并以替代实现的互操作测试确认。 | **条件通过，可进入 T002/T003。** 实施仍须以实际 Gradle 解析图、Debug 编译和 QR 互操作测试复核；这些尚未执行，不能写成 Android 运行时已验证。 |
| `net.dongliu:apk-parser` | `2.6.10`，BSD-2-Clause（白名单内） | 仅在 `:core:adb` 的可替换内部 APK 元数据适配器解析 label、资源和图标；项目公开模型不得暴露其类型。 | 发布 POM 的 `jsr305:3.0.0`、`bcprov-jdk15on:1.58`、`bcpkix-jdk15on:1.58` 均为 `optional=true`，`junit:4.12` 为 test scope；可选项不随 Gradle 运行时图传递。默认不显式加入它们；若实现确实需要其中任一项，必须先逐项完成白名单/许可证审查、固定版本和依赖图复核，不能把 POM optional 当作已批准。 | 上游 `hsiafan/apk-parser` 已归档，维护风险**高**。替代为项目自有受限 Manifest/resources parser（或重新 Plan 选择已维护且重新审查通过的实现）；移除时删除内部适配器、catalog alias 和 `:core:adb` implementation，保留项目自有模型与有界 reader。 | **高风险、仅条件接入；US4 当前 BLOCKED。** 在受控实现实际通过恶意 ZIP 防护（含 traversal、压缩炸弹、损坏 ZIP 与资源边界）和 Android API 30 runtime smoke test 前，不得解除。任一门禁失败：US4 保持 `BLOCKED`，返回 Plan 选择替代 parser；不得以整个 `UNSUPPORTED` 或仅包名占位计作应用名/图标需求完成。 |

### 未执行证据与结论边界

- 本任务未运行 Android API 30 runtime smoke test、恶意 APK/ZIP 测试、Gradle 解析或 APK 体积测量；这些均为 `NOT_RUN` 的后续实现门禁，不能从 POM 或样例解析推断为通过。
- 功能完成、设备验证和发布合规是独立结论。即使 ZXing 与 apk-parser 后续门禁均通过，`spake2-java 1.0.5` 的 GPL-3.0-or-later 问题仍使发布合规为 **BLOCKED**，直至 ADR 0004 关闭。
