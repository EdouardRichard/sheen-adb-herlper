## 1. 实际实现

### `:core:adb`

- 新增项目自有 `RemoteApplication`、`ApplicationSnapshot`、启用状态、可选字段和携带 session ID 的变更结果模型。
- `AdbSessionManager` 新增当前用户第三方应用列表、单包强停和单包启用状态修改的命名能力。
- 应用命令集中在内部能力层；当前用户和每条包管理命令均明确限定用户，不向 Feature/UI 暴露原始命令或第三方 ADB 类型。
- 实现 `am`/`cmd activity` 当前用户探测和 `pm`/`cmd package` 列表兼容路径；解析异常不返回半列表。
- 包名采用保守语法和 255 字符上限校验，列表限制 20,000 个去重项；包名必须来自同 session 最近安全快照并在操作前重新验证。
- 系统/范围外/状态未知目标保守拒绝；本机 session 拒绝操作 `com.sheen.adbhelper`。
- 状态修改执行前置范围验证、命名命令和后置状态验证；强停只返回“请求已接受”语义。
- 实现应用管理互斥、15/10/15 秒默认超时、取消与掉线清理、结果未知、session 切换防污染及细分错误映射。
- 诊断事件只记录应用阶段、结果、技术代码和脱敏端点，不记录包名、用户 ID、命令或输出。

### `:feature:apps`

- 新增独立 Compose Feature 模块，只依赖现有 `:core:adb`、`:core:ui`、Lifecycle、Coroutines 和 Compose 能力。
- 新增不可变 `AppsUiState`、ViewModel 和页面，覆盖离线、加载、内容、空、降级、错误、操作中、取消和结果未知。
- 包名搜索大小写不敏感；全部/已启用/已禁用筛选只作用于内存快照。
- 强停、禁用和重新启用每次都生成一次性确认；确认展示完整包名、设备显示名、当前用户与对应风险，取消按钮取得默认焦点。
- 加载/修改期间禁用其他管理入口并提供取消；离开页面取消工作和清除确认，session 变化清空列表、筛选、通知和旧结果。
- 不显示推测名称或图标，不持久化包列表、搜索、筛选、确认或操作结果。

### `:app`

- 注册 `:feature:apps` 并装配 `AppsViewModel`。
- 抽屉顺序更新为首页、概览、应用管理、Shell、进程、Logcat、设置。
- 应用管理离线可见但置灰并带读屏禁用语义；断开时从该目的地返回首页。
- 将匹配当前连接档案的设备显示名提供给应用确认状态。
- 版本更新为 `0.0.2 (2)`。

### 真机回归修复：Shell 与 Logcat

- 根因一：命令子流读取或关闭阶段的异常曾被 `executeShell` 升级为关闭整个活跃 session；Kadb 子流的已知 EOF/关闭异常也可能落入通用未知错误。现由 `core:adb` 显式持有每次命令的子流句柄，在 `finally` 仅关闭该句柄；成功、非零退出、超时、取消和命令级失败均不改变有效连接，用户命令字符串原样传给协议层。
- 根因二：Kadb 连接原使用 30 秒 Socket 读超时，持续 Logcat 在无新数据时可能被当作结束。连接级空闲读超时现关闭，有限时操作仍由 `AdbSessionManager` 的协程超时约束；Logcat 的停止、取消、断开和 session 切换分别按子流与连接所有权清理。
- 新增 `ADB_COMMAND_STREAM_CLOSED` 与 `ADB_IO_FAILURE`，Shell/Logcat 的已知子流 EOF、关闭和 I/O 异常不再统一映射为 `ADB_UNKNOWN`，且命令级错误不会写坏全局连接状态。
- Logcat 继续使用 `core:adb` 的前台 `shell-v2` 持续流；Feature 只保留进程内原始环形缓冲，并以过滤后的增量滚动窗口展示最新 100 条。暂停冻结可见窗口，继续时追上最新窗口，清屏后仍可接收新日志。
- 自动采集不落盘、不上传，也不把日志内容写入诊断事件；保留 v0.01 已有、必须由用户主动选择目标位置的可见内容导出能力。
- 后续回归确认用户容易把电脑端 `adb shell` 包装写法输入已经建立 ADB Session 的设备端 Shell。现由 `:core:adb` 只负责识别该语法，Feature 必须展示将执行的设备端内容并让用户明确选择；普通输入和“仍原样执行”路径保持逐字符不变，确认移除包装的记录在当前内存会话中可见。
- Logcat 启动后追赶设备缓冲区历史导致较晚到达当前尾部的问题不在 v0.02 修改；已登记 [`ADR 0006`](adr/0006-logcat-recent-history-follow-mode.md)，目标版本为 v0.03。

## 4. 已接受限制与发布边界

- 项目负责人已确认 v0.02 真机验证执行完成并批准正式结项；仓库不保存真实设备、地址、包名、Shell 输出、Logcat 或配对材料。
